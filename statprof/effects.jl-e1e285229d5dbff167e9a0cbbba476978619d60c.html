4 Represents computational effects of a method call.
5
6 The effects are a composition of different effect bits that represent some program property
7 of the method being analyzed. They are represented as `Bool` or `UInt8` bits with the
8 following meanings:
9 - `effects.consistent::UInt8`:
10 * `ALWAYS_TRUE`: this method is guaranteed to return or terminate consistently.
11 * `ALWAYS_FALSE`: this method may be not return or terminate consistently, and there is
12 no need for further analysis with respect to this effect property as this conclusion
13 will not be refined anyway.
14 * `CONSISTENT_IF_NOTRETURNED`: the `:consistent`-cy of this method can later be refined to
15 `ALWAYS_TRUE` in a case when the return value of this method never involves newly
16 allocated mutable objects.
17 * `CONSISTENT_IF_INACCESSIBLEMEMONLY`: the `:consistent`-cy of this method can later be
18 refined to `ALWAYS_TRUE` in a case when `:inaccessiblememonly` is proven.
19 - `effect_free::UInt8`:
20 * `ALWAYS_TRUE`: this method is free from externally semantically visible side effects.
21 * `ALWAYS_FALSE`: this method may not be free from externally semantically visible side effects, and there is
22 no need for further analysis with respect to this effect property as this conclusion
23 will not be refined anyway.
24 * `EFFECT_FREE_IF_INACCESSIBLEMEMONLY`: the `:effect-free`-ness of this method can later be
25 refined to `ALWAYS_TRUE` in a case when `:inaccessiblememonly` is proven.
26 - `nothrow::Bool`: this method is guaranteed to not throw an exception.
27 - `terminates::Bool`: this method is guaranteed to terminate.
28 - `notaskstate::Bool`: this method does not access any state bound to the current
29 task and may thus be moved to a different task without changing observable
30 behavior. Note that this currently implies that `noyield` as well, since
31 yielding modifies the state of the current task, though this may be split
32 in the future.
33 - `inaccessiblememonly::UInt8`:
34 * `ALWAYS_TRUE`: this method does not access or modify externally accessible mutable memory.
35 This state corresponds to LLVM's `inaccessiblememonly` function attribute.
36 * `ALWAYS_FALSE`: this method may access or modify externally accessible mutable memory.
37 * `INACCESSIBLEMEM_OR_ARGMEMONLY`: this method does not access or modify externally accessible mutable memory,
38 except that it may access or modify mutable memory pointed to by its call arguments.
39 This may later be refined to `ALWAYS_TRUE` in a case when call arguments are known to be immutable.
40 This state corresponds to LLVM's `inaccessiblemem_or_argmemonly` function attribute.
41 - `nonoverlayed::Bool`: indicates that any methods that may be called within this method
42 are not defined in an [overlayed method table](@ref OverlayMethodTable).
43 - `noinbounds::Bool`: If set, indicates that this method does not read the parent's `:inbounds`
44 state. In particular, it does not have any reached `:boundscheck` exprs, not propagates inbounds
45 to any children that do.
46
47 Note that the representations above are just internal implementation details and thus likely
48 to change in the future. See [`Base.@assume_effects`](@ref) for more detailed explanation
49 on the definitions of these properties.
50
51 Along the abstract interpretation, `Effects` at each statement are analyzed locally and they
52 are merged into the single global `Effects` that represents the entire effects of the
53 analyzed method (see the implementation of `merge_effects!`). Each effect property is
54 initialized with `ALWAYS_TRUE`/`true` and then transitioned towards `ALWAYS_FALSE`/`false`.
55 Note that within the current flow-insensitive analysis design, effects detected by local
56 analysis on each statement usually taint the global conclusion conservatively.
57
58 ## Key for `show` output of Effects:
59
60 The output represents the state of different effect properties in the following order:
61
62 1. `consistent` (`c`):
63 - `+c` (green): `ALWAYS_TRUE`
64 - `-c` (red): `ALWAYS_FALSE`
65 - `?c` (yellow): `CONSISTENT_IF_NOTRETURNED` and/or `CONSISTENT_IF_INACCESSIBLEMEMONLY`
66 2. `effect_free` (`e`):
67 - `+e` (green): `ALWAYS_TRUE`
68 - `-e` (red): `ALWAYS_FALSE`
69 - `?e` (yellow): `EFFECT_FREE_IF_INACCESSIBLEMEMONLY`
70 3. `nothrow` (`n`):
71 - `+n` (green): `true`
72 - `-n` (red): `false`
73 4. `terminates` (`t`):
74 - `+t` (green): `true`
75 - `-t` (red): `false`
76 5. `notaskstate` (`s`):
77 - `+s` (green): `true`
78 - `-s` (red): `false`
79 6. `inaccessiblememonly` (`m`):
80 - `+m` (green): `ALWAYS_TRUE`
81 - `-m` (red): `ALWAYS_FALSE`
82 - `?m` (yellow): `INACCESSIBLEMEM_OR_ARGMEMONLY`
83 7. `noinbounds` (`i`):
84 - `+i` (green): `true`
85 - `-i` (red): `false`
86
87 Additionally, if the `nonoverlayed` property is false, a red prime symbol (â€²) is displayed after the tuple.
