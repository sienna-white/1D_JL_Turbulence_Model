91 mutable struct TwoPhaseDefUseMap
92
93 This struct is intended as a memory- and GC-pressure-efficient mechanism
94 for incrementally computing def-use maps. The idea is that the def-use map
95 is constructed into two passes over the IR. In the first, we simply count the
96 the number of uses, computing the number of uses for each def as well as the
97 total number of uses. In the second pass, we actually fill in the def-use
98 information.
99
100 The idea is that either of these two phases can be combined with other useful
101 work that needs to scan the instruction stream anyway, while avoiding the
102 significant allocation pressure of e.g. allocating an array for every SSA value
103 or attempting to dynamically move things around as new uses are discovered.
104
105 The def-use map is presented as a vector of vectors. For every def, indexing
106 into the map will return a vector of uses.
