166 """
167 struct LimitedAccuracy
168
169 A `LimitedAccuracy` lattice element is used to indicate that the true inference
170 result was approximate due to heuristic termination of a recursion. For example,
171 consider two call stacks starting from `A` and `B` that look like:
172
173 A -> C -> A -> D
174 B -> C -> A -> D
175
176 In the first case, inference may have decided that `A->C->A` constitutes a cycle,
177 widening the result it obtained for `C`, even if it might otherwise have been
178 able to obtain a result. In this case, the result inferred for `C` will be
179 annotated with this lattice type to indicate that the obtained result is an
180 upper bound for the non-limited inference. In particular, this means that the
181 call stack originating at `B` will re-perform inference without being poisoned
182 by the potentially inaccurate result obtained during the inference of `A`.
183
184 N.B.: We do *not* take any efforts to ensure the reverse. For example, if `B`
185 is inferred first, then we may cache a precise result for `C` and re-use this
186 result while inferring `A`, even if inference of `A` would have not been able
187 to obtain this result due to limiting. This is undesirable, because it makes
188 some inference results order dependent, but there it is unclear how this situation
189 could be avoided.
190
191 A `LimitedAccuracy` element wraps another lattice element (let's call it `T`)
192 and additionally tracks the `causes` due to which limitation occurred. As a
193 lattice element, `LimitedAccuracy(T)` is considered ε smaller than the
194 corresponding lattice element `T`, but in particular, all lattice elements that
195 are `⊑ T` (but not equal `T`) are also `⊑ LimitedAccuracy(T)`.
196
197 The `causes` list is used to determine whether a particular cause of limitation is
198 inevitable and if so, widening `LimitedAccuracy(T)` back to `T`. For example,
199 in the call stacks above, if any call to `A` always leads back to `A`, then
200 it does not matter whether we start at `A` or reach it via `B`: Any inference
201 that reaches `A` will always hit the same limitation and the result may thus
202 be cached.
203 """
