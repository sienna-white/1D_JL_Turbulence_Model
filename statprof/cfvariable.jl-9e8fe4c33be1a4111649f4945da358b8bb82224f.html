5 """
6 defVar(ds::NCDataset,name,vtype,dimnames; kwargs...)
7 defVar(ds::NCDataset,name,data,dimnames; kwargs...)
8
9 Define a variable with the name `name` in the dataset `ds`. `vtype` can be
10 Julia types in the table below (with the corresponding NetCDF type). The
11 parameter `dimnames` is a tuple with the names of the dimension. For scalar
12 this parameter is the empty tuple `()`.
13 The variable is returned (of the type `CFVariable`).
14
15 Instead of providing the variable type one can directly give also the data `data` which
16 will be used to fill the NetCDF variable. In this case, the dimensions with
17 the appropriate size will be created as required using the names in `dimnames`.
18
19 If `data` is a vector or array of `DateTime` objects, then the dates
20 are saved as double-precision floats and units
21 "$(CFTime.DEFAULT_TIME_UNITS)" (unless a time unit
22 is specifed with the `attrib` keyword as described below). Dates are
23 converted to the default calendar in the CF conversion which is the
24 mixed Julian/Gregorian calendar.
25
26 ## Keyword arguments
27
28 * `fillvalue`: A value filled in the NetCDF file to indicate missing data.
29 It will be stored in the _FillValue attribute. `NCDatasets` does not use implicitely the default NetCDF fill values when reading data.
30 * `chunksizes`: Vector integers setting the chunk size. The total size of a chunk must be less than 4 GiB.
31 * `deflatelevel`: Compression level: 0 (default) means no compression and 9 means maximum compression. Each chunk will be compressed individually.
32 * `shuffle`: If true, the shuffle filter is activated which can improve the compression ratio.
33 * `checksum`: The checksum method can be `:fletcher32` or `:nochecksum` (checksumming is disabled, which is the default)
34 * `attrib`: An iterable of attribute name and attribute value pairs, for example a `Dict`, `DataStructures.OrderedDict` or simply a vector of pairs (see example below)
35 * `typename` (string): The name of the NetCDF type required for [vlen arrays](https://web.archive.org/save/https://www.unidata.ucar.edu/software/netcdf/netcdf-4/newdocs/netcdf-c/nc_005fdef_005fvlen.html)
36
37 `chunksizes`, `deflatelevel`, `shuffle` and `checksum` can only be
38 set on NetCDF 4 files. Compression of strings and variable-length arrays is not
39 supported by the underlying NetCDF library.
40
41 ## NetCDF data types
42
43 | NetCDF Type | Julia Type |
44 |-------------|------------|
45 | NC_BYTE | Int8 |
46 | NC_UBYTE | UInt8 |
47 | NC_SHORT | Int16 |
48 | NC_INT | Int32 |
49 | NC_INT64 | Int64 |
50 | NC_FLOAT | Float32 |
51 | NC_DOUBLE | Float64 |
52 | NC_CHAR | Char |
53 | NC_STRING | String |
54
55
56 ## Dimension ordering
57
58 The data is stored in the NetCDF file in the same order as they are stored in
59 memory. As julia uses the
60 [Column-major ordering](https://en.wikipedia.org/wiki/Row-_and_column-major_order)
61 for arrays, the order of dimensions will appear reversed when the data is loaded
62 in languages or programs using
63 [Row-major ordering](https://en.wikipedia.org/wiki/Row-_and_column-major_order)
64 such as C/C++, Python/NumPy or the tools `ncdump`/`ncgen`
65 ([NetCDF CDL](https://web.archive.org/web/20220513091844/https://docs.unidata.ucar.edu/nug/current/_c_d_l.html)).
66 NumPy can also use Column-major ordering but Row-major order is the default. For the column-major
67 interpretation of the dimensions (as in Julia), the
68 [CF Convention recommends](https://web.archive.org/web/20220328110810/http://cfconventions.org/Data/cf-conventions/cf-conventions-1.7/cf-conventions.html#dimensions) the
69 order "longitude" (X), "latitude" (Y), "height or depth" (Z) and
70 "date or time" (T) (if applicable). All other dimensions should, whenever
71 possible, be placed to the right of the spatiotemporal dimensions.
72
73 ## Example:
74
75 In this example, `scale_factor` and `add_offset` are applied when the `data`
76 is saved.
77
78 ```julia-repl
79 julia> using DataStructures
80 julia> data = randn(3,5)
81 julia> NCDataset("test_file.nc","c") do ds
82 defVar(ds,"temp",data,("lon","lat"), attrib = OrderedDict(
83 "units" => "degree_Celsius",
84 "add_offset" => -273.15,
85 "scale_factor" => 0.1,
86 "long_name" => "Temperature"
87 ))
88 end;
89 ```
90
91 !!! note
92
93 If the attributes `_FillValue`, `missing_value`, `add_offset`, `scale_factor`,
94 `units` and `calendar` are used, they should be defined when calling `defVar`
95 by using the parameter `attrib` as shown in the example above.
