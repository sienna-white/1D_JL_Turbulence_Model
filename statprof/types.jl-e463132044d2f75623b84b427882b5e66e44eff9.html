93 """
94 inf_params::InferenceParams
95
96 Parameters that control abstract interpretation-based type inference operation.
97
98 ---
99 - `inf_params.max_methods::Int = 3`\\
100 Type inference gives up analysis on a call when there are more than `max_methods` matching
101 methods. This trades off between compiler latency and generated code performance.
102 Typically, considering many methods means spending _lots_ of time obtaining poor type
103 information, so this option should be kept low. [`Base.Experimental.@max_methods`](@ref)
104 can have a more fine-grained control on this configuration with per-module or per-method
105 annotation basis.
106 ---
107 - `inf_params.max_union_splitting::Int = 4`\\
108 Specifies the maximum number of union-tuples to swap or expand before computing the set of
109 matching methods or conditional types.
110 ---
111 - `inf_params.max_apply_union_enum::Int = 8`\\
112 Specifies the maximum number of union-tuples to swap or expand when inferring a call to
113 `Core._apply_iterate`.
114 ---
115 - `inf_params.max_tuple_splat::Int = 32`\\
116 When attempting to infer a call to `Core._apply_iterate`, abort the analysis if the tuple
117 contains more than this many elements.
118 ---
119 - `inf_params.tuple_complexity_limit_depth::Int = 3`\\
120 Specifies the maximum depth of large tuple type that can appear as specialized method
121 signature when inferring a recursive call graph.
122 ---
123 - `inf_params.ipo_constant_propagation::Bool = true`\\
124 If `false`, disables analysis with extended lattice information, i.e. disables any of
125 the concrete evaluation, semi-concrete interpretation and constant propagation entirely.
126 [`Base.@constprop :none`](@ref Base.@constprop) can have a more fine-grained control on
127 this configuration with per-method annotation basis.
128 ---
129 - `inf_params.aggressive_constant_propagation::Bool = false`\\
130 If `true`, forces constant propagation on any methods when any extended lattice
131 information available. [`Base.@constprop :aggressive`](@ref Base.@constprop) can have a
132 more fine-grained control on this configuration with per-method annotation basis.
133 ---
134 - `inf_params.unoptimize_throw_blocks::Bool = true`\\
135 If `true`, skips inferring calls that are in a block that is known to `throw`.
136 It may improve the compiler latency without sacrificing the runtime performance
137 in common situations.
138 ---
139 - `inf_params.assume_bindings_static::Bool = false`\\
140 If `true`, assumes that no new bindings will be added, i.e. a non-existing binding at
141 inference time can be assumed to always not exist at runtime (and thus e.g. any access to
142 it will `throw`). Defaults to `false` since this assumption does not hold in Julia's
143 semantics for native code execution.
144 ---
145 """

218 opt_params::OptimizationParams
219
220 Parameters that control optimizer operation.
221
222 ---
223 - `opt_params.inlining::Bool = inlining_enabled()`\\
224 Controls whether or not inlining is enabled.
225 ---
226 - `opt_params.inline_cost_threshold::Int = 100`\\
227 Specifies the number of CPU cycles beyond which it's not worth inlining.
228 ---
229 - `opt_params.inline_nonleaf_penalty::Int = 1000`\\
230 Specifies the penalty cost for a dynamic dispatch.
231 ---
232 - `opt_params.inline_tupleret_bonus::Int = 250`\\
233 Specifies the extra inlining willingness for a method specialization with non-concrete
234 tuple return types (in hopes of splitting it up). `opt_params.inline_tupleret_bonus` will
235 be added to `opt_params.inline_cost_threshold` when making inlining decision.
236 ---
237 - `opt_params.inline_error_path_cost::Int = 20`\\
238 Specifies the penalty cost for an un-optimized dynamic call in a block that is known to
239 `throw`. See also [`(inf_params::InferenceParams).unoptimize_throw_blocks`](@ref InferenceParams).
240 ---
241 - `opt_params.max_tuple_splat::Int = 32`\\
242 When attempting to inline `Core._apply_iterate`, abort the optimization if the tuple
243 contains more than this many elements.
244 ---
245 - `opt_params.compilesig_invokes::Bool = true`\\
246 If `true`, gives the inliner license to change which `MethodInstance` to invoke when
247 generating `:invoke` expression based on the [`@nospecialize`](@ref) annotation,
248 in order to avoid over-specialization.
249 ---
250 - `opt_params.trust_inference::Bool = false`\\
251 If `false`, the inliner will unconditionally generate a fallback block when union-splitting
252 a callsite, in case of existing subtyping bugs. This option may be removed in the future.
253 ---
254 - `opt_params.assume_fatal_throw::Bool = false`\\
255 If `true`, gives the optimizer license to assume that any `throw` is fatal and thus the
256 state after a `throw` is not externally observable. In particular, this gives the
257 optimizer license to move side effects (that are proven not observed within a particular
258 code path) across a throwing call. Defaults to `false`.
259 ---

349 function NativeInterpreter(world::UInt = get_world_counter();
350 inf_params::InferenceParams = InferenceParams(),
351 opt_params::OptimizationParams = OptimizationParams())
352 # Sometimes the caller is lazy and passes typemax(UInt).
353 # we cap it to the current world age for correctness
354 if world == typemax(UInt)
355 world = get_world_counter()
356 end
357
358 # If they didn't pass typemax(UInt) but passed something more subtly
359 # incorrect, fail out loudly.
360 @assert world <= get_world_counter()
361
362 method_table = CachedMethodTable(InternalMethodTable(world))
363
364 inf_cache = Vector{InferenceResult}() # Initially empty cache
365
366 return NativeInterpreter(world, method_table, inf_cache, inf_params, opt_params, #=irinterp=#false)
367 end
