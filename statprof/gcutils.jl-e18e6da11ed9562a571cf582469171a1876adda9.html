4 """
5 WeakRef(x)
6
7 `w = WeakRef(x)` constructs a [weak reference](https://en.wikipedia.org/wiki/Weak_reference)
8 to the Julia value `x`: although `w` contains a reference to `x`, it does not prevent `x` from being
9 garbage collected. `w.value` is either `x` (if `x` has not been garbage-collected yet) or `nothing`
10 (if `x` has been garbage-collected).
11
12 ```jldoctest
13 julia> x = "a string"
14 "a string"
15
16 julia> w = WeakRef(x)
17 WeakRef("a string")
18
19 julia> GC.gc()
20
21 julia> w # a reference is maintained via `x`
22 WeakRef("a string")
23
24 julia> x = nothing # clear reference
25
26 julia> GC.gc()
27
28 julia> w
29 WeakRef(nothing)
30 ```

45 """
46 finalizer(f, x)
47
48 Register a function `f(x)` to be called when there are no program-accessible references to
49 `x`, and return `x`. The type of `x` must be a `mutable struct`, otherwise the function
50 will throw.
51
52 `f` must not cause a task switch, which excludes most I/O operations such as `println`.
53 Using the `@async` macro (to defer context switching to outside of the finalizer) or
54 `ccall` to directly invoke IO functions in C may be helpful for debugging purposes.
55
56 Note that there is no guaranteed world age for the execution of `f`. It may be
57 called in the world age in which the finalizer was registered or any later world age.
58
59 # Examples
60 ```julia
61 finalizer(my_mutable_struct) do x
62 @async println("Finalizing \$x.")
63 end
64
65 finalizer(my_mutable_struct) do x
66 ccall(:jl_safe_printf, Cvoid, (Cstring, Cstring), "Finalizing %s.", repr(x))
67 end
68 ```
69
70 A finalizer may be registered at object construction. In the following example note that
71 we implicitly rely on the finalizer returning the newly created mutable struct `x`.
72
73 # Example
74 ```julia
75 mutable struct MyMutableStruct
76 bar
77 function MyMutableStruct(bar)
78 x = new(bar)
79 f(t) = @async println("Finalizing \$t.")
80 finalizer(f, x)
81 end
82 end
83 ```
84 """
85 function finalizer(@nospecialize(f), @nospecialize(o))
86 _check_mutable(o)
87 Core.finalizer(f, o)
88 return o
89 end

168 Returns `true` if the current task is running a finalizer, returns `false`

182 """
183 GC.@preserve x1 x2 ... xn expr
184
185 Mark the objects `x1, x2, ...` as being *in use* during the evaluation of the
186 expression `expr`. This is only required in unsafe code where `expr`
187 *implicitly uses* memory or other resources owned by one of the `x`s.
188
189 *Implicit use* of `x` covers any indirect use of resources logically owned by
190 `x` which the compiler cannot see. Some examples:
191 * Accessing memory of an object directly via a `Ptr`
192 * Passing a pointer to `x` to `ccall`
193 * Using resources of `x` which would be cleaned up in the finalizer.
194
195 `@preserve` should generally not have any performance impact in typical use
196 cases where it briefly extends object lifetime. In implementation, `@preserve`
197 has effects such as protecting dynamically allocated objects from garbage
198 collection.
199
200 # Examples
201
202 When loading from a pointer with `unsafe_load`, the underlying object is
203 implicitly used, for example `x` is implicitly used by `unsafe_load(p)` in the
204 following:
205
206 ```jldoctest
207 julia> let
208 x = Ref{Int}(101)
209 p = Base.unsafe_convert(Ptr{Int}, x)
210 GC.@preserve x unsafe_load(p)
211 end
212 101
213 ```
214
215 When passing pointers to `ccall`, the pointed-to object is implicitly used and
216 should be preserved. (Note however that you should normally just pass `x`
217 directly to `ccall` which counts as an explicit use.)
218
219 ```jldoctest
220 julia> let
221 x = "Hello"
222 p = pointer(x)
223 Int(GC.@preserve x @ccall strlen(p::Cstring)::Csize_t)
224 # Preferred alternative
225 Int(@ccall strlen(x::Cstring)::Csize_t)
226 end
227 5
228 ```
229 """
230 macro preserve(args...)
231 syms = args[1:end-1]
232 for x in syms
233 isa(x, Symbol) || error("Preserved variable must be a symbol")
234 end
235 esc(Expr(:gc_preserve, args[end], syms...))
236 end

239 GC.safepoint()
240
241 Inserts a point in the program where garbage collection may run.
242 This can be useful in rare cases in multi-threaded programs where some threads
243 are allocating memory (and hence may need to run GC) but other threads are doing
244 only simple operations (no allocation, task switches, or I/O).
245 Calling this function periodically in non-allocating threads allows garbage
246 collection to run.

248 !!! compat "Julia 1.4"
249 This function is available as of Julia 1.4.
