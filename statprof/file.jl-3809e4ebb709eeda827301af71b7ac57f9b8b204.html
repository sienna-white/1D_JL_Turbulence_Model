35 CSV.File(input; kwargs...) => CSV.File
36
37 Read a UTF-8 CSV input and return a `CSV.File` object, which is like a lightweight table/dataframe, allowing dot-access to columns
38 and iterating rows. Satisfies the Tables.jl interface, so can be passed to any valid sink, yet to avoid unnecessary copies of data,
39 use `CSV.read(input, sink; kwargs...)` instead if the `CSV.File` intermediate object isn't needed.
40
41 The [`input`](@ref input) argument can be one of:
42 * filename given as a string or FilePaths.jl type
43 * a `Vector{UInt8}` or `SubArray{UInt8, 1, Vector{UInt8}}` byte buffer
44 * a `CodeUnits` object, which wraps a `String`, like `codeunits(str)`
45 * a csv-formatted string can also be passed like `IOBuffer(str)`
46 * a `Cmd` or other `IO`
47 * a gzipped file (or gzipped data in any of the above), which will automatically be decompressed for parsing
48 * a `Vector` of any of the above, which will parse and vertically concatenate each source, returning a single, "long" `CSV.File`
49
50 To read a csv file from a url, use the Downloads.jl stdlib or HTTP.jl package, where the resulting downloaded tempfile or `HTTP.Response` body can be passed like:
51 ```julia
52 using Downloads, CSV
53 f = CSV.File(Downloads.download(url))
54
55 # or
56
57 using HTTP, CSV
58 f = CSV.File(HTTP.get(url).body)
59 ```
60
61 Opens the file or files and uses passed arguments to detect the number of columns and column types, unless column types are provided
62 manually via the `types` keyword argument. Note that passing column types manually can slightly increase performance
63 for each column type provided (column types can be given as a `Vector` for all columns, or specified per column via
64 name or index in a `Dict`).
65
66 When a `Vector` of inputs is provided, the column names and types of each separate file/input must match to be vertically concatenated. Separate threads will
67 be used to parse each input, which will each parse their input using just the single thread. The results of all threads are then vertically concatenated using
68 `ChainedVector`s to lazily concatenate each thread's columns.
69
70 For text encodings other than UTF-8, load the [StringEncodings.jl](https://github.com/JuliaStrings/StringEncodings.jl)
71 package and call e.g. `CSV.File(open(read, input, enc"ISO-8859-1"))`.
72
73 The returned `CSV.File` object supports the [Tables.jl](https://github.com/JuliaData/Tables.jl) interface
74 and can iterate `CSV.Row`s. `CSV.Row` supports `propertynames` and `getproperty` to access individual row values. `CSV.File`
75 also supports entire column access like a `DataFrame` via direct property access on the file object, like `f = CSV.File(file); f.col1`.
76 Or by getindex access with column names, like `f[:col1]` or `f["col1"]`. The returned columns are `AbstractArray` subtypes, including:
77 `SentinelVector` (for integers), regular `Vector`, `PooledVector` for pooled columns, `MissingVector` for columns of all `missing` values,
78 `PosLenStringVector` when `stringtype=PosLenString` is passed, and `ChainedVector` will chain one of the previous array types together for
79 data inputs that use multiple threads to parse (each thread parses a single "chain" of the input).
80 Note that duplicate column names will be detected and adjusted to ensure uniqueness (duplicate column name `a` will become `a_1`).
81 For example, one could iterate over a csv file with column names `a`, `b`, and `c` by doing:
82
83 ```julia
84 for row in CSV.File(file)
85 println("a=\$(row.a), b=\$(row.b), c=\$(row.c)")
86 end
87 ```
88
89 By supporting the Tables.jl interface, a `CSV.File` can also be a table input to any other table sink function. Like:
90
91 ```julia
92 # materialize a csv file as a DataFrame, copying columns from CSV.File
93 df = CSV.File(file) |> DataFrame
94
95 # to avoid making a copy of parsed columns, use CSV.read
96 df = CSV.read(file, DataFrame)
97
98 # load a csv file directly into an sqlite database table
99 db = SQLite.DB()
100 tbl = CSV.File(file) |> SQLite.load!(db, "sqlite_table")
