13
14 Construct a `SentinelArray` by either wrapping an existing `AbstractArray` or using the standard `undef` pattern, which first constructs an `Array` to be wrapped by `SentinelArray`.
15
16 A `SentinelArray{T}` wraps an `AbstractArray` of type `T`, and accepts a `sentinel` and `value` argument. If the wrapped array has an element equal to `sentinel`, `value` is returned instead.
17 This allows, for example, simulating a `Vector{Union{Float64, Missing}}` by doing something like `SentinelVector{Float64}(undef, len, NaN, missing)`. The resulting `SentinelArray` will have
18 `eltype` equal to `Union{T, typeof(value)}`.
19
20 For isbits types, a random/reasonable sentinel will be attempted if none provided. For non-isbits, `#undef` is used as sentinel. The default value, if not provided, is `missing`.
21
22 A `SentinelArray` allows setting an element to `value`, which will result in the `sentinel` being stored in the wrapped array.
23
24 For isbits element types, if a `setindex!` operation is attempted with the `SentinelArray`'s current sentinel value, another sentinel will be attempted to be chosen that doesn't already exist in the wrapped array.
25 If another sentinel can't be found (i.e. the values already exist in the wrapped array), an error will be thrown.
26
27 **CAVEATS**:
28 * For float and date/time types, reasonable sentinels exist that don't have meaningful *real* values, so it's recommended to use the default sentinels
29 * For `Integer` types, all bit patterns are valid, so a random value is chosen; if usage will involve extremely high cardinality relative to the type range, consider using `Vector{Union{T, Missing}}` instead
30 * `SentinelArray`s are not allowed to have plain `Bool` elements since there isn't a possible sentinel
31 * A `SentinelArray` *IS NOT* thread-safe when usage relies on `setindex!` and potential sentinel collision (due to the check to automatically cycle the sentinel to another value); this will normally only affect small `Integer` element types where the potential for sentinel collision is highest
32 * Access to a `SentinelArray`'s raw data is possible via `parent(A)` or `pointer(A)`, but note that the raw data will *NOT* respect the sentinel=>value semantics; i.e. the raw data may have entries of the sentinel value.
33
34 Use-cases:
35 * It is generally recommended to use `Vector{Union{T, typeof(value}}` instead of `SentinelArray` if possible
36 * Currently, a non-copying operation like `convert(Vector{T}, A::Vector{Union{T, Missing}})` isn't possible, even though you may know there aren't `missing` values; a `SentinelArray` allows this operation by just "unwrapping" the storage array, like `parent(A)`
37 * It's also currently not possible to use `Vector{Union{T, Missing}}` with `Mmap.mmap`; with `SentinelArray`, you can write the parent array to disk and then do `SentinelArray(Mmap.mmap(array_type, file))` (assuming usage of default sentinel/value)
38
39 Examples:
40 ```julia
41 # initialize a new SentinelArray with default sentinel/value
42 A = SentinelVector{Float64}(undef, 10)
43
44 # wrap an existing array, manually providing sentinel/value
45 parent = zeros(UInt8, 10)
46 B = SentinelArray(parent, typemax(UInt8), nothing)
47 ```
